# Godot Minigame 3

## Devlog

This lesson encouraged me to code the way I normally do. I normally write a line or two of code, then check to see if it worked. I do this because it's easier to find problems in my code. If I test the lines of code and they don't work, I know there is a problem. If I were to wait and test my code after I wrote everything, I would have no clue where the problem is. I think my method is helping me because if something doesn't work, I know where the problem is without having to spend time looking for it. I think my process is working well for me; it makes coding easier.
